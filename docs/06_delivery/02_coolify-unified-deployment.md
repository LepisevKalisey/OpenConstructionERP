# Инструкция по развертыванию OpenConstructionERP через Dockerfile.unified в Coolify

Данное руководство описывает пошаговый процесс деплоя OpenConstructionERP в режиме "всё в одном" (Unified Image) с использованием базы данных SQLite и векторного хранилища LanceDB. Это наиболее экономичный по ресурсам вариант, идеально подходящий для виртуальных серверов (VPS) с небольшим объемом оперативной памяти.

- **Статус**: Активен
- **Тип документа**: How-to (инструкция)
- **Язык**: Русский
- **Дата**: 2026-05-23
- **Автор**: Antigravity (DevOps-инженер)

---

## Предварительные требования

1. Доступ к панели Coolify (например, `https://c00l.lexi.kz/`).
2. Доступ к вашему форку репозитория: `LepisevKalisey/OpenConstructionERP`.
3. Сгенерированный секретный ключ для JWT токенов (необходим для авторизации пользователей).
   * Вы можете сгенерировать его на своем компьютере командой:
     ```bash
     openssl rand -hex 32
     ```
     *(Или использовать любую случайную строку длиной 64 символа).*

## Пошаговая настройка в интерфейсе Coolify

### Шаг 1: Создание нового приложения (Application)
1. Откройте панель Coolify, перейдите в ваш проект и выберите окружение (например, **Production**).
2. Нажмите кнопку **+ Add Resource** и выберите **Public Repository** или **Private Repository** (в зависимости от настроек видимости вашего форка).
3. Укажите ссылку на ваш репозиторий: `https://github.com/LepisevKalisey/OpenConstructionERP`.
4. Укажите ветку (обычно `main` или `master`).
5. Нажмите **Save**.

### Шаг 2: Настройка сборщика (Build Pack)
После создания приложения откройте его настройки и перейдите во вкладку **Configuration**:
1. Найдите поле **Build Pack** и измените его значение на **Dockerfile**.
2. В поле **Dockerfile Path** укажите относительный путь к унифицированному Docker-файлу в репозитории:
   ```text
   deploy/docker/Dockerfile.unified
   ```
3. Сохраните изменения.

### Шаг 3: Настройка сетевого порта
1. Во вкладке конфигурации сети найдите параметр **Ports / Destination Port**.
2. Установите порт назначения: **`8080`** (это стандартный порт, на котором запускается веб-сервер в `Dockerfile.unified`).
3. Настройте публичный домен (FQDN), например: `https://erp.lexi.kz` (Coolify автоматически выпустит для него SSL-сертификат от Let's Encrypt).

### Шаг 4: Монтирование дискового пространства (Storage / Volumes)
Чтобы база данных SQLite и загруженные файлы проектов (чертежи, BIM-модели) не стирались при каждом обновлении приложения, нужно настроить постоянный том:
1. Перейдите во вкладку **Storage** или найдите раздел **Persistent Storage (Volumes)**.
2. Добавьте новое монтирование:
   * **Volume Name**: `openestimate-data` (или любое другое имя)
   * **Destination Path (внутри контейнера)**: `/data`
3. Нажмите кнопку сохранения. Это смонтирует папку `/data` контейнера на постоянный диск сервера.

### Шаг 5: Переменные окружения (Environment Variables)
Перейдите во вкладку **Environment Variables** и добавьте следующие переменные:

| Ключ переменной | Значение | Описание |
| :--- | :--- | :--- |
| `JWT_SECRET` | *[Ваш сгенерированный hex-ключ]* | **Обязательно.** Используется для шифрования пользовательских сессий. |
| `SERVE_FRONTEND` | `true` | Говорит бэкенду раздавать собранный React-интерфейс. |
| `APP_ENV` | `production` | Переводит приложение в режим продакшена. |
| `DATABASE_URL` | `sqlite+aiosqlite:////data/openestimate.db` | Путь к файлу базы данных SQLite внутри контейнера. |
| `DATABASE_SYNC_URL` | `sqlite:////data/openestimate.db` | Путь для синхронных миграций БД Alembic. |
| `VECTOR_BACKEND` | `lancedb` | Локальный файловый векторный движок (не требует запуска Qdrant). |
| `VECTOR_DATA_DIR` | `/data/vectors` | Папка для хранения векторов семантического поиска. |
| `ALLOWED_ORIGINS` | `*` | Разрешенные адреса для CORS запросов. |

## Запуск деплоя

1. В правом верхнем углу страницы приложения нажмите кнопку **Deploy**.
2. Coolify начнет сборку:
   * **Этап 1:** Установка Node.js зависимостей и сборка React-фронтенда в оптимизированный `dist`.
   * **Этап 2:** Сборка Python-образа, установка необходимых pip-пакетов (включая зависимости для работы с сервером).
   * **Этап 3:** Копирование фронтенда в бэкенд-пакет, запуск веб-сервера Uvicorn.
3. Дождитесь успешного завершения сборки. Статус приложения должен смениться на `Healthy`.
